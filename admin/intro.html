🤖 CommunityOS Lite 打开控制台

轻量多 Bot Telegram 控制台

本机 FastAPI + 长轮询 getUpdates， 统一配置大模型与文本知识库，几分钟搭好社群自动应答。

进入 Lite 控制台 Agent Skill 安装说明 GitHub 仓库

核心能力

🤖 多 Bot 管理

创建、启用/停用、分别配置 Token、人设（Soul）与知识片段，适合多社群、多品牌分身。

✨ 全局 LLM

MiniMax、OpenAI、Claude、DeepSeek 等统一在「LLM」页配置；所有 Bot 共用一套推理后端，按需切换模型。

📚 文本知识库

无需向量库，直接把说明、FAQ 贴进知识文本，回答会优先贴合你提供的范围。

💬 群聊自动应答

开启被动问答后，群内文本可触发回复（需结合 Telegram 群隐私设置，见下方说明）。

🔒 私聊开关

按 Bot 控制是否接受私信，避免运营号被私聊淹没。

🏠 仅本机管理

默认绑定 127.0.0.1，适合本机或内网使用；请勿将控制台端口直接暴露公网。

📵 Telegram 群隐私（群里「没反应」时先看这里）

新机器人默认开启 Group Privacy。开启时，群里往往只有命令、回复机器人、@机器人用户名 的消息会推送到本服务。

若希望普通聊天也能触发自动回复：在 @BotFather → /mybots → 选你的 Bot → Bot Settings → Group Privacy → Turn off。

EN: New bots default to privacy ON. For plain group messages to reach Lite, turn Group Privacy OFF in BotFather. Private chats are unaffected.

与完整 CommunityOS 的区别

Lite（本页产品） 单进程、JSON 配置、无 Harness 治理引擎；适合快速上线、单机运维。
完整版 含 harness.yaml、多 Bot 协作与治理组件；需要脚手架与协作编排时再选用完整栈。

CommunityOS Lite · Agent Skill 安装

⚠️ 请先区分产品

本 Skill 只说明本仓库里 Telegram Lite 控制台的部署与排错。与微信 / 小程序等 Agent Skill 不是同一产品；请勿与 panda-wechat-skill 等目录混放、嵌套或随意改名。

方式一：一键安装（推荐）

  1. 在 Cursor / OpenClaw 终端执行：
    clawhub install community-os-lite

    自动安装到 skills 目录，无需手动复制。

  2. 重启 Agent，Skill 即生效。

方式二：手动安装

  1. 在克隆下来的本仓库中找到 Skill 目录：
    来源（仓库内）
    .cursor/skills/community-os-lite/
  2. 将整个 community-os-lite 文件夹复制到你的目标工程：
    目标（你的项目）
    你的项目/.cursor/skills/community-os-lite/

    其他编辑器若 skills 根路径不同，只要保证最后一级文件夹名仍为 community-os-lite 即可。
